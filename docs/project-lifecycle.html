Sproutflow Studio — Internal Reference

Project Lifecycle Architecture

The end-to-end system used from contract signature through long-term client partnership — every phase, gate, deliverable, and client touchpoint.

🤝
01
Kickoff

Project Kickoff

Days 1–3 after contract signing

Align on goals, constraints, success metrics, and communication norms before any work begins. Ambiguity surfaced here costs nothing. Ambiguity discovered in build costs weeks.

Key Activities
Kickoff call (60–90 min) Intake questionnaire review Project brief creation Stakeholder alignment Tool access & credentials Communication cadence set
Deliverables
  • Signed project brief (goals, audience, success KPIs)
  • Finalized sitemap / page list
  • Project board set up (tasks, milestones, timeline)
  • Shared asset folder with brand files
Gate: Brief approved by client before design begins
🎨
02
Design

Design & Creative Direction

Week 1–2 (varies by package)

Translate strategy into a visual system. The goal is not just aesthetics — it's creating a layout logic that drives conversion, trust, and clarity at every scroll depth.

Key Activities
Moodboard / style direction Wireframes (key pages) Hi-fi mockups in Figma Responsive breakpoints Copy placeholder / structure Design review rounds (2x)
Deliverables
  • Approved moodboard / direction
  • Wireframes for all key pages
  • High-fidelity desktop + mobile designs (Figma)
  • Component inventory (buttons, cards, nav, footer)
Gate: Final design sign-off before any code is written
⚙️
03
Build

Development & Build

Week 2–4 (varies by scope)

Convert approved designs into a performant, maintainable codebase. Staging environment is live throughout — no surprises on launch day. Real content is loaded as it becomes available.

Key Activities
Dev environment setup Component build (top-down) CMS integration Forms & integrations Real content load-in Performance & SEO baseline Cross-browser QA Accessibility pass
Deliverables
  • Staging URL with full site build
  • Responsive across mobile, tablet, desktop
  • Lighthouse scores documented (performance, a11y, SEO)
  • All forms tested and integrated
  • Content fully loaded (no placeholder text)
Gate: Client staging review + QA sign-off before launch
🚀
04
Launch

Launch & Handoff

Day of launch + 48-hr stabilization window

White-glove go-live. DNS migration, cache warming, and post-launch monitoring handled by the studio. Client is trained and handed full ownership with confidence, not confusion.

Key Activities
Pre-launch checklist run DNS cutover SSL verification Analytics verified Search Console submitted 48-hr monitoring window Client walkthrough/training
Deliverables
  • Live site confirmed fully functional
  • Credentials & access doc handed off securely
  • Client training session (recorded)
  • Site documentation (how-to edits, CMS guide)
  • Project retrospective notes (internal)
🚀 Live — 48-hr monitoring period begins immediately
🌱
05
Maintenance

Ongoing Care & Growth

Monthly — ongoing partnership

The site is a living asset, not a finished artifact. Care plans protect uptime, performance, and security while creating a structured channel for iteration and expansion over time.

Care Plan Tiers
Basic — updates + uptime monitoring Growth — + content changes + monthly report Partner — + priority support + iteration sprints
Recurring Activities
  • Plugin / dependency updates & security patches
  • Uptime and performance monitoring
  • Content edits and page additions (by tier)
  • Monthly analytics report with insights
  • Quarterly strategy check-in (Partner tier)
  • Conversion audits and A/B iteration (Partner tier)
Watch for Escalation Triggers
Performance regression Security vulnerability Traffic spike / infra limit Major rebrand signal

How to Read This Architecture

Phase CardsTimeline flows left-to-right alternating. Each card covers one phase with activities, deliverables, and an exit gate.

Gates (🔒)Hard stops requiring explicit client approval. Work on the next phase cannot begin until the gate is cleared.

Deliverables (✓)Tangible outputs committed per phase. These are what the client receives and what the studio is accountable for.

Client TouchpointsThe bottom strip maps every scheduled client interaction per phase to prevent surprise and set expectations.

Escalation PillsOrange pills in Maintenance flag conditions that should prompt proactive client outreach or scope conversation.
